Cross-Cutting: API Limitations for AI/ML

Key Gaps Affecting All AI/ML Use Cases

  • No webhooks — must poll for new/changed data. Real-time ML triggers require periodic sync.
  • No server-side aggregation — all sums, averages, and distributions computed client-side.
  • No audit log — no "who changed what" data. Limits anomaly root-cause analysis.
  • No budget data — cannot compare actual vs planned. Limits anomaly context.
  • Rate limits undocumented — HTTP 429 returned at unknown threshold. Build with exponential backoff.
  • No feedback API — ML models can't learn from user corrections via the API. Build feedback loop externally.

Mitigation Strategy

  • Use data_alteracao_de/ate for incremental sync instead of full pulls
  • Cache categories, cost centers, and pessoa data locally (changes rarely)
  • Set quantidade_registros=1000 on financial endpoints to minimize pages
  • Build your own feedback/correction layer outside the Conta Azul API
