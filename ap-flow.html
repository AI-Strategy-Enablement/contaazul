Detailed Coverage Breakdown

StepCoverageAPI SpecKey EndpointsNotes
1. Supplier Onboarding Full Person API POST /v1/pessoas GET /v1/pessoas PATCH /v1/pessoas/{id} Create suppliers with "Supplier" (Fornecedor) profile, CNPJ/CPF, addresses, state registrations. Batch activate/deactivate supported.
2. Purchase Order Gap — — No PO management API exists. The ERP tracks purchases internally (COMPRA origin in payment records) but exposes no purchase API.
3. Supplier Invoice Receipt Gap — — Invoice API only covers issued invoices (sales). ERP has internal document import (IMPORTACAO_DOCUMENTO, NOTA_COMPRA origins in payment records) but no API for it. Attachments schema (AnexoParcela) supports FATURA type but is read-only.
4. 3-Way Matching Gap — — No validation/matching logic exposed. Product master data available via Inventory API for reference only.
5. AP Recording Full Financial API POST /v1/financeiro/.../contas-a-pagar GET .../contas-a-pagar/buscar PATCH .../parcelas/{id} Full CRUD for payable events. Supports installments, categories, cost centers, financial accounts.
6. Approval Workflow Gap — — No approval workflow or authorization state management for payments.
7. Payment Scheduling Partial Financial API PATCH .../parcelas/{id} GET /v1/conta-financeira/{id}/saldo-atual Due dates and scheduled payment flag exist. No dedicated scheduling management API.
8. Payment Execution Partial Financial API PATCH .../parcelas/{id} Payments recorded via write-offs ("baixas"). Supports partial payments, PIX, transfer, boleto. No dedicated endpoint.
9. Bank Reconciliation Partial Financial API GET /v1/financeiro/transferencias Reconciliation IDs exist in baixa records. Transfers queryable. No dedicated reconciliation workflow.
10. Financial Reporting Full Financial API GET /v1/categorias GET /v1/centro-de-custo GET /v1/financeiro/categorias-dre Full category hierarchy, cost centers, P&L (DRE) mapping. 123 default categories available.
