Accounts Payable Flow

AP Detailed Coverage Breakdown

StepCoverageAPI SpecKey EndpointsNotes
1. Supplier Onboarding Full Person API POST /v1/pessoas GET /v1/pessoas PATCH /v1/pessoas/{id} Create suppliers with "Supplier" (Fornecedor) profile, CNPJ/CPF, addresses, state registrations. Batch activate/deactivate supported.
2. Purchase Order Gap — — No PO management API exists. The ERP tracks purchases internally (COMPRA origin in payment records) but exposes no purchase API.
3. Supplier Invoice Receipt Gap — — Invoice API only covers issued invoices (sales). ERP has internal document import (IMPORTACAO_DOCUMENTO, NOTA_COMPRA origins) but no API. Attachments schema (AnexoParcela) supports FATURA type but is read-only.
4. 3-Way Matching Gap — — No validation/matching logic exposed. Product master data available via Inventory API for reference only.
5. AP Recording Full Financial API POST /v1/financeiro/.../contas-a-pagar GET .../contas-a-pagar/buscar PATCH .../parcelas/{id} Full CRUD for payable events. Supports installments, categories, cost centers, financial accounts.
6. Approval Workflow Gap — — No approval workflow or authorization state management for payments.
7. Payment Scheduling Partial Financial API PATCH .../parcelas/{id} GET /v1/conta-financeira/{id}/saldo-atual Due dates and scheduled payment flag exist. No dedicated scheduling management API.
8. Payment Execution Partial Financial API PATCH .../parcelas/{id} Payments recorded via write-offs ("baixas"). Supports partial payments, PIX, transfer, boleto. No dedicated endpoint.
9. Bank Reconciliation Partial Financial API GET /v1/financeiro/transferencias Reconciliation IDs exist in baixa records. Transfers queryable. No dedicated reconciliation workflow. Deep dive →
10. Financial Reporting Full Financial API GET /v1/categorias GET /v1/centro-de-custo GET /v1/financeiro/categorias-dre Full category hierarchy, cost centers, P&L (DRE) mapping. 123 default categories available.
Accounts Receivable Flow

AR Detailed Coverage Breakdown

StepCoverageAPI SpecKey EndpointsNotes
1. Customer Onboarding Full Person API POST /v1/pessoas PATCH /v1/pessoas/{id} Full CRUD for customers with billing contact field (contato_cobranca_faturamento). Batch activate/deactivate.
2. Quote / Sales Order Full Sales API POST /v1/venda PUT /v1/venda/{id} GET /v1/venda/busca Full draft-to-approved flow: EM_ANDAMENTO (draft) → APROVADO → FATURADO. Line items with products/services, payment conditions, installments.
3. Invoice Issuance Partial Sales + Invoice API PUT /v1/venda/{id} (set FATURADO) GET /v1/notas-fiscais No direct invoice creation API. Setting sale to FATURADO triggers NFe/NFS-e in ERP. Invoice API is read-only but shows pre-issuance states (PENDENTE, PRONTA_ENVIO).
4. Invoice Delivery Full Sales + Invoice API GET /v1/venda/{id}/imprimir GET /v1/notas-fiscais/{chave} PDF generation from sales. XML retrieval by access key for issued invoices.
5. AR Recording Full Financial API POST .../contas-a-receber GET .../contas-a-receber/buscar PATCH .../parcelas/{id} Full CRUD for receivables. Async creation (HTTP 202). Installments with 7 statuses.
6. Billing & Collection Partial Financial API SolicitacaoCobranca schema in Parcela Rich collection system: BOLETO, LINK_PAGAMENTO, PIX_COBRANCA. Payment URLs generated. But no dedicated creation endpoint.
7. Cash Receipt Full Financial API PATCH .../parcelas/{id} Payments via write-offs ("baixas") with 22+ payment methods. Partial payments supported. Digital receipts tracked.
8. Bank Reconciliation Partial Financial API GET /v1/financeiro/transferencias id_reconciliacao in Baixa records. conciliado flag on Parcela. No dedicated reconciliation workflow. Deep dive →
9. Recurring Billing Full Contracts API POST /v1/contratos GET /v1/contratos Create recurring contracts with Monthly/Annual (MENSAL/ANUAL) frequency. Auto-generates sales per schedule.
10. Financial Reporting Full Financial API GET /v1/categorias GET /v1/centro-de-custo GET /v1/financeiro/categorias-dre 123 financial categories, cost centers, P&L (DRE) mapping.

Invoice Attachment & Document Capabilities

FeatureStatusDetails
Invoice PDF Generation Available GET /v1/venda/{id}/imprimir returns binary PDF
Invoice XML Retrieval Available GET /v1/notas-fiscais/{chave} returns XML by access key
AnexoParcela (Installment Attachments) Read-only Types: FATURA, BOLETO_BANCARIO, RECIBO, RECIBO_DIGITAL, OUTROS. Supports FILE and URL content. No upload endpoint.
AnexoBaixa (Payment Attachments) Read-only Types: RECIBO_DIGITAL, RECIBO. System-generated when payment recorded.
SolicitacaoCobranca (Collection Docs) Read-only System generates boleto PDFs, payment links, PIX QR codes. URL returned in response.
Direct Invoice Creation (NFe/NFS-e) Not Available Invoice API explicitly states read-only. Creation triggered indirectly via sale status FATURADO.
Draft Invoice States Visible, not writable NFS-e statuses include PENDENTE and PRONTA_ENVIO (pre-issuance). Queryable but not settable via API.
