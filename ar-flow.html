Detailed AR Coverage Breakdown

StepCoverageAPI SpecKey EndpointsNotes
1. Customer Onboarding Full Person API POST /v1/pessoas PATCH /v1/pessoas/{id} Full CRUD for customers with billing contact field (contato_cobranca_faturamento). Batch activate/deactivate.
2. Quote / Sales Order Full Sales API POST /v1/venda PUT /v1/venda/{id} GET /v1/venda/busca Full draft-to-approved flow: EM_ANDAMENTO (draft) → APROVADO → FATURADO. Line items with products/services, payment conditions, installments.
3. Invoice Issuance Partial Sales + Invoice API PUT /v1/venda/{id} (set FATURADO) GET /v1/notas-fiscais No direct invoice creation API. Setting sale to FATURADO triggers NFe/NFS-e in ERP. Invoice API is read-only but shows pre-issuance states (PENDENTE, PRONTA_ENVIO). NFS-e has 14 lifecycle statuses.
4. Invoice Delivery Full Sales + Invoice API GET /v1/venda/{id}/imprimir GET /v1/notas-fiscais/{chave} PDF generation from sales. XML retrieval by access key for issued invoices. Both product (NFe) and service (NFS-e) queryable.
5. AR Recording Full Financial API POST .../contas-a-receber GET .../contas-a-receber/buscar PATCH .../parcelas/{id} Full CRUD for receivables. Async creation (HTTP 202). Installments with 7 statuses (PENDENTE, QUITADO, ATRASADO, RECEBIDO_PARCIAL, etc.).
6. Billing & Collection Partial Financial API SolicitacaoCobranca schema in Parcela Rich collection system: BOLETO, LINK_PAGAMENTO, PIX_COBRANCA, BOLETO_REGISTRADO. Payment URLs generated. 12+ collection statuses. But no dedicated creation endpoint — collection requests appear as nested data in parcela responses.
7. Cash Receipt Full Financial API PATCH .../parcelas/{id} Payments via write-offs ("baixas") with 22+ payment methods (PIX, boleto, card, transfer, etc.). Partial payments supported. Digital receipts tracked (id_recibo_digital).
8. Bank Reconciliation Partial Financial API GET /v1/financeiro/transferencias id_reconciliacao in Baixa records. conciliado flag on Parcela. No dedicated reconciliation workflow endpoint.
9. Recurring Billing Full Contracts API POST /v1/contratos GET /v1/contratos Create recurring contracts with Monthly/Annual (MENSAL/ANUAL) frequency. Auto-generates sales per schedule. Returns id_venda for initial sale. Payment due day configurable.
10. Financial Reporting Full Financial API GET /v1/categorias GET /v1/centro-de-custo GET /v1/financeiro/categorias-dre 123 financial categories, cost centers, P&L (DRE) mapping. AR search supports aging by due date, customer, status, amount range.

Invoice Attachment & Document Capabilities

FeatureStatusDetails
Invoice PDF Generation Available GET /v1/venda/{id}/imprimir returns binary PDF
Invoice XML Retrieval Available GET /v1/notas-fiscais/{chave} returns XML by access key
AnexoParcela (Installment Attachments) Read-only Types: FATURA, BOLETO_BANCARIO, RECIBO, RECIBO_DIGITAL, OUTROS. Supports FILE and URL content. No upload endpoint.
AnexoBaixa (Payment Attachments) Read-only Types: RECIBO_DIGITAL, RECIBO. System-generated when payment recorded.
SolicitacaoCobranca (Collection Docs) Read-only System generates boleto PDFs, payment links, PIX QR codes. URL returned in response. No manual creation endpoint.
Direct Invoice Creation (NFe/NFS-e) Not Available Invoice API explicitly states read-only. Creation triggered indirectly via sale status FATURADO.
Draft Invoice States Visible, not writable NFS-e statuses include PENDENTE and PRONTA_ENVIO (pre-issuance). These are queryable but not settable via API.
